# Data Files

Expected workflow:

1. `data/raw/hackernews_comments.jsonl`
   - Generated by `uv run takemeter-collect-hn`.
   - Contains public HN comments with empty labels.

2. `data/labeled/takemeter_hn_review_queue.jsonl`
   - Generated by `uv run python -m ai201_project3_takemeter.heuristic_prelabler`.
   - Contains rough labels and must be manually reviewed.

3. `data/labeled/takemeter_hn_labeled.jsonl`
   - Final reviewed source dataset.
   - Must contain at least 200 rows.
   - Required keys: `text`, `label`.
   - Useful extra keys: `source`, `story_title`, `notes`, `review_status`.

4. `data/colab/takemeter_hn_labeled_for_colab.csv`
   - Generated from the reviewed JSONL.
   - Temporary adapter for the course starter Colab notebook.
   - Contains only `text,label`.
   - Ignored by git; JSONL remains the repo source dataset.

Validate the final JSONL dataset:

```bash
uv run takemeter-validate-dataset data/labeled/takemeter_hn_labeled.jsonl
```

For the rough review queue only, add `--allow-unreviewed` so the command checks
count and distribution without pretending the labels are final.

Export the Colab CSV:

```bash
uv run takemeter-export-colab-csv
```
